## 2. 아카이브 페이지 UI 구조

### 2.1 레이아웃
- **Learning Queue 벤치마크** 기반 2분할 레이아웃
- 왼쪽: 글 리스트 (카드형)
- 오른쪽: 선택한 글의 상세 내용
- 모바일: 세로 스택 (리스트 위, 상세 아래)

### 2.2 3탭 시스템

| 탭 | 역할 | 비유 |
|---|---|---|
| **수집 (inbox)** | 작업 끝나면 일단 여기에 쌓임. 가공 전 원재료 | 이메일 받은편지함 |
| **선별 (shortlist)** | "블로그에 올릴만하다" 판단된 글 | 이메일 중요표시 |
| **발행완료 (published)** | 블로그에 실제 발행된 글. 원본 보관용 | 이메일 보낸편지함 |

### 2.3 기능 목록 (프로토타입에 구현 완료)

| 기능 | 상태 | 설명 |
|---|---|---|
| 3탭 (수집/선별/발행완료) | ✅ | 탭 간 이동 버튼 포함 |
| 소스 컬러 바 | ✅ | 카드 왼쪽 3px 세로선 (소스별 색상) |
| 소스 아이콘 뱃지 | ✅ | 💬 claude.ai, ⌨️ claude code, 🎬 lilys ai, 📝 memo |
| 멀티 태그 필터 | ✅ | 여러 태그 동시 선택 (OR 조건) |
| 검색 | ✅ | 제목+태그+본문 통합 검색 |
| 검색 하이라이트 | ✅ | 매칭 키워드 노란색 강조 |
| 정렬 토글 | ✅ | 최신순 ↔ 오래된순 |
| 태그 편집 | ✅ | 상세 패널에서 태그 추가(+)/삭제(×) |
| AI 요약 섹션 | ✅ | 각 글의 요약 표시 |
| 원본 링크 | ✅ | Claude 대화, GitHub, Lilys AI 바로가기 |
| 메모 필드 | ✅ | 각 글에 개인 메모 작성 |
| 다크/라이트 모드 | ✅ | 다크: 소프트 차콜 톤, 라이트: 크림/베이지 톤 |
| 상태 이동 | ✅ | 수집→선별→발행 버튼 |

## 4. 태그 선정 기준

태그는 **"나중에 이 글을 어떤 키워드로 찾을까?"** 기준으로 붙인다.

| 태그 유형 | 예시 | 용도 |
|---|---|---|
| 프로젝트명 | `CruiseAI`, `사라요가`, `kangs-blog` | 어떤 프로젝트 작업인지 |
| 기술/도구 | `Playwright`, `Gemma4`, `ClaudeCode`, `GitHubPages` | 어떤 기술을 썼는지 |
| 작업 유형 | `크롤링`, `디자인`, `설계`, `디버깅`, `배포` | 뭘 했는지 |
| 주제 | `AI전환`, `비용최적화`, `멀티에이전트` | 어떤 주제인지 |

- 한 글에 **3~6개**가 적당
- 너무 적으면 검색 안 됨, 너무 많으면 필터 의미 없음

## 5. 데이터 구조 (archive.json)

```json
{
  "id": 1,
  "status": "inbox",
  "source": "claude-ai",
  "date": "2026-04-11",
  "title": "제목",
  "excerpt": "한줄 요약 (카드에 표시)",
  "summary": "AI가 정리한 3~4줄 요약 (상세 패널 요약 섹션)",
  "body": "본문 HTML (<h4>, <p>, <ul>, <li>, <code> 사용)",
  "tags": ["태그1", "태그2", "태그3"],
  "links": {
    "claude": "https://claude.ai/chat/xxx",
    "github": "https://github.com/xxx",
    "lilys": "https://lilys.ai/digest/xxx",
    "blog": "https://galincia-hub.github.io/kangs-blog/blog/xxx.html"
  },
  "memo": "개인 메모 (선택)"
}
```

### 필드 설명

| 필드 | 타입 | 필수 | 설명 |
|---|---|---|---|
| `id` | number | ✅ | 고유 ID. 기존 최대값 + 1 |
| `status` | string | ✅ | `inbox` / `shortlist` / `published` |
| `source` | string | ✅ | `claude-ai` / `claude-code` / `lilys` / `memo` |
| `date` | string | ✅ | ISO 날짜 (YYYY-MM-DD) |
| `title` | string | ✅ | 글 제목 |
| `excerpt` | string | ✅ | 1~2줄 요약 (리스트 카드에 표시) |
| `summary` | string | ✅ | 3~4줄 AI 요약 (상세 패널에 표시) |
| `body` | string | ✅ | 본문 HTML |
| `tags` | array | ✅ | 태그 배열 (3~6개 권장) |
| `links` | object | ✅ | 원본 링크들. 키: claude, github, lilys, blog |
| `memo` | string | - | 개인 메모 |

## 6. 등록 트리거 설계

### 6.1 Claude Code 작업 완료 시 (자동)

CLAUDE.md에 아래 규칙을 추가:

```
작업 완료 후 반드시 archive/archive.json에 항목을 추가하고 git push:
- id: 기존 최대 id + 1
- status: "inbox"
- source: "claude-code"
- date: 오늘 날짜 (YYYY-MM-DD)
- title: 작업 제목
- excerpt: 한줄 요약
- summary: 3~4줄 요약 (이번 작업에서 뭘 했고, 뭐가 막혔고, 어떻게 해결했는지)
- body: 본문 HTML
- tags: 관련 태그 3~6개
- links: { "github": "커밋/PR URL" }
- memo: ""
```

### 6.2 Claude.ai 대화 후 (반자동)

대화 끝나면 "아카이브에 추가해줘"라고 요청.
→ archive.json에 넣을 JSON을 생성
→ Claude Code에서 한 줄 명령으로 추가 + git push

### 6.3 Lilys AI (반자동)

Lilys AI가 노션에 저장한 데이터를 Notion API로 읽어서 변환.
- 노션 위치: "Lilys AI" 데이터베이스 (ID: `33fab334-c1c0-8178-b191-e289cd2c7e66`)
- 하위 페이지: "자세한 리포트", "타임라인 요약" 등

### 6.4 수동 메모 (수동)

아카이브 페이지 내 입력 폼에서 직접 작성 (추후 구현).

## 8. 배포 방침

- **아카이브는 비공개** — 블로그 nav에 링크 넣지 않음
- 경로: `kangs-blog/archive/index.html`
- URL 아는 사람만 접근 가능 (GitHub Pages는 비밀번호 보호 불가)
- `robots.txt`에 `/archive/` 차단 추가 (검색엔진 노출 방지)
- `sitemap.xml`에 아카이브 URL 포함하지 않음
- 아카이브 페이지 내에 **"블로그 바로가기"** 링크만 존재

## 11. 프로토타입 파일

프로토타입 HTML 파일(`archive-v2.html`)은 이 프로젝트의 파일에 포함되어 있습니다.
- 샘플 데이터 10개 포함 (실제 작업 내용 기반)
- 모든 UI 기능 동작 확인 가능
- 이 파일을 `archive/index.html`로 배포하면 됨
- 배포 시 하드코딩된 데이터를 `archive.json`으로 분리하는 작업 필요

## 12. 벤치마크 참고

### Learning Queue (장피엠)
- 3단계 탭: Later → Shortlist → Archive
- 2분할 레이아웃 (리스트 + 상세)
- 태그 수평 나열 + 클릭 필터
- AI 요약 자동 생성
- "클로드 코드로 보내기" 버튼

### Algotraction
- 라이트모드 색상 참고 (크림/베이지 톤)

### 차별점
- 아카이브 → 블로그 발행 → SNS 변환까지 이어지는 파이프라인
- Claude.ai 대화 링크 통합 (검색/필터 보완)
- 소스별 컬러 바로 시각적 구분
