AI 활용기 크루즈 인사이트

Week 3: AI가 쓴 글이 SEO+AEO 테스트를 통과한 순간

Pagery 2026년 3월 16일
#SEO #AEO #CruiseStudio

이번 주에 뭘 했나

3월 10일부터 16일. 이번 주는 배포가 많았다. CruiseMD 프로모션 랜딩 템플릿 + CLI 생성, Streamlit Cloud 배포(cruisemd.streamlit.app), Notion MCP 연동(Claude Code → Notion 작업일지 자동 생성), blog-preview GitHub Pages 배포 — 33개 크루즈 블로그 포스트가 실제로 온라인에 올라갔다.

그리고 3/15에 SEO+AEO 통합 콘텐츠 마스터 가이드라인 v1.0을 만들고, CruiseStudio에 적용했다. 테스트 결과: 30개 항목 중 FAIL 0.

어디서 막혔나

막힌 건 기술 문제가 아니었다. 60% 완성된 시점에서 "이게 맞는 방향인가" 하는 의심이 왔다. 콘텐츠를 계속 만들어야 하는지, 아니면 사이트 구조를 먼저 완성해야 하는지. "콘텐츠 vs 구조" 갈팡질팡이었다.

AEO가 뭔지도 이 시점에 처음 제대로 알았다. SEO는 알고 있었다 — 검색 엔진 최적화. AEO는 Answer Engine Optimization. AI 검색엔진(ChatGPT, Perplexity 등)에서 답변으로 인용되도록 콘텐츠를 최적화하는 것. 크루즈 업계에서 이게 앞으로 중요해진다는 걸 여기서 처음 정리했다.

어떻게 넘겼나

전체 시스템 전략을 점검했다. Phase A(수익 파이프라인) / B(콘텐츠 엔진) / C(고객 루프) 3단계로 정리하고, 지금 단계에서 해야 할 것 하나만 골랐다: "가장 작은 완성 하나를 만드는 것."

그게 SEO+AEO 가이드라인이었다. 10개 섹션짜리 문서를 만들고, CruiseStudio에 적용하고, 30개 체크리스트로 테스트했다. FAIL 0이 나왔을 때 — "아, 이게 동작하는구나"라는 확신이 생겼고, 다음 단계를 밀어붙일 수 있었다.

가오픈 전략도 이 주에 확정했다: "완성 후 오픈"이 아니라 "오픈하면서 완성."

뭘 배웠나

"가장 작은 완성 하나를 만드는 것이 피로감을 이기는 유일한 방법."

50대에 새로운 걸 배우면서 가장 힘든 게 피로감이다. "이게 될까?" 하는 의심. 근데 30개 FAIL 0을 보는 순간 그게 사라진다. 숫자가 주는 확신은 말로 설득하는 것보다 강하다.
