Week 1: CruiseStudio 파이프라인의 시작

Pagery 2026년 2월 11일
#CruiseAI #빌드인퍼블릭 #여행업AI

이번 주에 뭘 했나

구정 연휴를 이용해서 오래 머릿속에 있던 실험을 시작했다. 목표는 간단했다: 유튜브 크루즈 영상의 스크립트를 Claude API에 넣으면 블로그 포스팅이 자동으로 나오게 하는 것.

거창하게 시작한 건 아니다. 일단 Claude Code로 Python 스크립트 하나를 만들었다. YouTube 링크 입력 → 자막 추출 → Claude API 호출 → HTML 초안 출력. 이게 이번 주 목표였고, 결론적으로 됐다.

어디서 막혔나

유튜브 자막 추출이 첫 번째 벽이었다. youtube-transcript-api 라이브러리를 쓰면 된다고 Claude가 알려줬는데, 한국어 자막이 없는 영상이 절반이었다. 영어 자막만 있으면 번역까지 해야 하는데, 그러면 파이프라인이 복잡해진다.

두 번째 문제는 Claude API 응답 품질이었다. 스크립트를 그냥 넣으면 너무 밋밋한 글이 나왔다. "크루즈 여행의 매력을 소개합니다" 같은 홍보문구 느낌. 내가 원하는 건 실제 정보성 콘텐츠인데.

어떻게 넘겼나

자막 문제는 일단 영어 자막 있는 영상으로 범위를 좁혔다. 완벽한 해결은 아니지만 일단 돌아가게 하는 게 먼저라고 판단했다.

프롬프트 품질 문제는 Claude Code랑 같이 30분 정도 씨름했다. 핵심은 컨텍스트였다. "크루즈 여행 전문 블로그 에디터 역할, 독자는 처음 크루즈를 고려하는 40-50대, 실용적인 정보 중심"으로 시스템 프롬프트를 바꾸니까 확연히 달라졌다.

프롬프트는 역할 + 독자 + 목적, 이 세 가지를 명확히 하면 결과가 바뀐다.

뭘 배웠나

완벽한 시스템보다 돌아가는 시스템이 먼저다. 한국어 자막 없는 영상 문제는 나중에 해결하면 된다. 일단 영어 자막 영상 100개로 테스트하는 게 지금 단계에서 맞다.
