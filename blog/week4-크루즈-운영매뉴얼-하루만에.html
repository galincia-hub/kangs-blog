크루즈 인사이트 작업일지

Week 4: 2,400명 크루즈 운영 매뉴얼을 하루 만에

Pagery 2026년 3월 23일
#코스타세레나 #크루즈운영 #AI활용

이번 주에 뭘 했나

3월 17일, 코스타 세레나 전세선 운영 매뉴얼을 만들었다. 숫자부터 말하면: 7종 파일, 28개 시트, 111개 테이블. 하루 만에.

코스타 세레나는 2,400명이 탑승하는 대형 크루즈선이다. 전세선 운영이면 그 승객 전체를 우리가 관리한다는 뜻이다. 총괄 마스터플랜, 조직도, 프로그램팀 매뉴얼, 지원팀 매뉴얼, 승하선 절차, 기항지 운영, 자문 질문 시트. 이 7가지가 하루에 나왔다.

그리고 그걸 그냥 문서로 두지 않고 웹사이트(cruise-ops-desk)로 만들어서 GitHub Pages에 배포했다. 전 직원이 링크 하나로 접근할 수 있게.

어디서 막혔나

처음에는 Claude Code로 직접 HTML을 만들려고 했다. 근데 28개 시트의 복잡한 테이블 구조를 Claude Code에게 바로 HTML로 만들라고 하면 — 일관성이 없어지고, 수정할 때마다 전체를 다시 봐야 한다. 코드가 지저분해진다.

28개 시트를 어떻게 구조화할 건지 스스로도 헷갈렸다. 엑셀로 만들까, 노션으로 만들까, 그냥 워드로 만들까.

어떻게 넘겼나

역할을 분리했다. claude.ai = 데이터 소스 제작, Claude Code = 프론트엔드 빌드.

먼저 claude.ai에서 각 시트의 내용을 만들고 엑셀로 정리했다. 그 다음 JSON으로 변환했다. 그리고 Claude Code에게 "이 JSON 구조를 읽어서 HTML 사이트를 만들어줘"라고 했다.

JSON이 중간 형식으로 들어가면서 모든 게 정돈됐다. Claude Code는 HTML 빌드에만 집중하면 됐다. 출력도 일관됐고, 수정도 쉬웠다. build.py 하나로 JSON → HTML 변환 자동화까지 완성.

뭘 배웠나

"7개 파일, 28개 시트, 111개 테이블"을 하루 만에 만든 건 AI 없이는 수주가 걸렸을 작업이다.

23년 동안 여행업에서 일하면서 이런 매뉴얼을 한두 번 본 적 있다. 외주를 주거나, 팀 전체가 달라붙어서 몇 주 걸렸다. 근데 나 혼자, 하루에, 웹사이트까지 나왔다. 이게 AI 활용의 실증이다. 거창한 이야기가 아니라, 오늘 당장 쓸 수 있는 것들이.
