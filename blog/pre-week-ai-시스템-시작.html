Pre-Week: AI로 여행사 사업 시스템을 만들기로 한 날

Pagery 2026년 2월 27일
#CruiseAI #빌드인퍼블릭 #Claude Code

이번 주에 뭘 했나

2월 초부터 2월 말까지 약 3주. 이 기간을 뭐라고 불러야 할지 모르겠어서 "Pre-Week"이라고 이름 붙였다. 아직 Claude Code를 본격적으로 쓰기 전이고, 노션 기반 AI 어시스턴트 시스템을 운영하던 시기다.

노션에 AI 워크플로우를 만들어 놓고 크루즈 업무를 보조하고 있었다. 큐시트(작업 계획서), Level 1/2/3 난이도 체계, 자가 점검 Q/A 형식까지 — 20개 이상의 규칙/스킬 문서를 꽤 공들여 구축했다. 여기에 더해 CruiseKMS 설계서 v1.0도 썼다. 4계층 아키텍처, MCP Server 6개 핵심 툴, 5단계 로드맵. 그리고 YouTube→Blog 에이전트 설계서도. Writer→Reviewer→Style→SEO 4단계 파이프라인.

설계는 꽤 그럴싸했다. 문제는 실행이었다.

어디서 막혔나

노션 AI 시스템이 생각보다 느렸다. 정확히는 — 내가 만든 시스템이 나를 느리게 만들고 있었다.

"큐시트 승인 후 작업 진행"이라는 절차가 있었다. 매번 작업 계획을 작성하고, 검토하고, 승인하고, 그다음에야 실행. 그 과정이 30분짜리 작업을 2시간짜리로 만들어버렸다. 정교한 시스템을 만드는 데 너무 많은 에너지를 쏟고 있었고, 실제로 만들어지는 건 별로 없었다.

23년 동안 여행업에서 일하면서 느낀 건 하나다. 완벽한 매뉴얼보다 실제로 돌아가는 절반짜리가 낫다. 그런데 AI 시스템은 왜 반대로 만들고 있었을까.

어떻게 넘겼나

Claude Code를 제대로 써보기로 했다. 정확히는 — 노션 AI 시스템을 버리고 Claude Code + CLAUDE.md 기반으로 전환하기로 결심했다.

CLAUDE.md는 코드 파일이다. 프로젝트 루트에 두면 Claude Code가 항상 읽는다. 여기에 규칙을 쓰면 매번 큐시트를 작성할 필요가 없다. 그냥 열고 시작하면 된다.

20년 전에 리눅스 웹마스터 교육을 받은 적이 있다. 제로보드로 홈페이지도 만들었다. 그 기억이 어딘가에 있었던 것 같다. VS Code 터미널을 열고 Claude Code를 설치하는 게 생각보다 무섭지 않았다.

뭘 배웠나

"완벽한 워크플로우를 설계한 후 실행"보다 "실행하면서 워크플로우를 다듬는" 접근이 더 빠르다.

50대가 되면 뭔가를 시작하기 전에 준비를 많이 하게 된다. 실패가 무섭고, 시간이 아까우니까. 그런데 AI는 반대다. 일단 해보면 된다. 틀리면 수정하면 된다. 20대에 제로보드 쓸 때 그렇게 했던 것처럼.
