Week 2: CLAUDE.md가 진짜 메모리다

Pagery 2026년 3월 9일
#CLAUDE.md #Git #AI협업

이번 주에 뭘 했나

이번 주(3/3~3/9)는 인프라 정비 주간이었다. Ruby/Jekyll 설치 후 로컬 빌드 확인, Netlify 배포, Streamlit 대시보드 5개 탭 구현. dev-hub GitHub Pages 배포도 이번 주다. 그리고 가장 큰 수확 — GitHub 백업 체계 구축. CruiseMD, CruiseStudio, CruiseKMS, CruiseDB 4개 리포지토리를 한꺼번에 GitHub에 올렸다.

겉으로 보면 "인프라 정비"인데, 사실 이번 주 핵심은 하나였다. Claude Code 프로젝트 메모리가 로컬 PC에만 저장된다는 걸 알게 된 것.

어디서 막혔나

나는 회사에서도 작업하고, 집에서도 작업한다. Claude Code의 프로젝트 메모리가 ~/.claude/ 폴더 — PC 로컬에 저장된다는 걸 모르고 있었다.

회사에서 Claude Code에게 "이 프로젝트는 이런 구조야, 이 규칙을 따라줘"라고 가르쳤다. 그리고 집에서 같은 프로젝트를 열었더니 Claude Code가 아무것도 모른다. 처음 본 프로젝트처럼 행동한다.

이걸 고치려고 처음엔 클라우드 동기화를 생각했다. 근데 ~/.claude/ 폴더를 동기화하면 여러 문제가 생길 수 있다. 더 근본적인 해결이 필요했다.

어떻게 넘겼나

CLAUDE.md 파일이 답이었다. 프로젝트 루트에 CLAUDE.md를 두면 Claude Code가 대화를 시작할 때 항상 읽는다. 이 파일을 Git으로 관리하면 — 회사에서 git push, 집에서 git pull — 두 환경에서 동일한 규칙을 쓸 수 있다.

여기서부터 루틴이 생겼다: 작업 시작 → git pull / 작업 종료 → git push. 이게 전부다. 클라우드 구독 필요 없고, 복잡한 동기화 도구 필요 없다.

CLAUDE.md에는 프로젝트 구조, 코딩 규칙, 금지사항, 자주 쓰는 명령어를 적어뒀다. 이게 사실상 AI와 나 사이의 "업무 계약서"다.

뭘 배웠나

"CLAUDE.md = 진짜 메모리. Git으로 동기화되는 이것이 회사/집 공유의 핵심."

AI는 기억을 못한다. 대화가 끝나면 끝이다. 근데 CLAUDE.md는 항상 남아있다. 파일이니까. 그래서 중요한 규칙은 대화로 가르치지 말고 파일에 써두는 게 맞다. 한 번 쓰면 어디서나 쓸 수 있다.
