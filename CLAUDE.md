# kangs-blog

## 토큰 효율 규칙
- 이미 읽은 파일은 같은 세션에서 다시 읽지 않는다
- 불필요한 도구 호출을 하지 않는다 (확인용 재조회 금지)
- 병렬 실행 가능한 도구 호출은 동시에 실행한다
- 20줄 이상의 반복 출력은 서브에이전트로 위임한다
- 사용자가 이미 설명한 내용을 반복하지 않는다

## 프로젝트 개요
- 역할: 개인 블로그 + 포트폴리오 사이트 (빌드 로그 형태)
- 기술 스택: 순수 HTML/CSS/JS + GitHub Pages
- 레퍼런스: 사라요가 프로젝트 (galincia-hub/sara)

## 핵심 원칙
- 200줄 이내 유지. 초과 시 별도 파일로 분리하고 참조만 남긴다
- 30분 이상 작업 시 /compact, 다른 작업 전환 시 /clear 권장
